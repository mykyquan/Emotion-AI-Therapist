# Contributing

Thanks for considering a contribution.

## Workflow

1. Fork the repository.
2. Create a focused branch for your change.
3. Make small, reviewable edits.
4. Do not commit secrets, `.env` files, Redis dumps, logs, or user data.
5. Run the relevant checks before opening a pull request:

```bash
npm run install:all
npm run build
npm run lint
```

6. Open a pull request with a clear description of what changed and how you tested it.

## Project Scope

This is an educational public beta starter kit. Contributions should keep the project beginner-friendly, reusable, and honest about safety and production limitations.
